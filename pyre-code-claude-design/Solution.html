Walkthrough

Causal self-attention is a one-line trick on top of scaled dot-product attention: mask scores above the diagonal with -inf before softmax. The resulting row i has zero probability mass on positions j > i, so token i's output depends only on itself and its past.

⚑ Core insight Mask before softmax, not after. If you zero out entries after softmax, the remaining entries no longer sum to 1, and gradients at masked positions become non-zero — which is how NaNs appear later in training.

Reference implementation

solutions / causal_attention.py 
import torch
import torch.nn.functional as F

def causal_attention(q, k, v):
    # q, k, v: (B, H, T, D)
    B, H, T, D = q.shape
    scores = q @ k.transpose(-2, -1) / D**0.5

    # build an upper-triangular mask: True = future = forbidden
    mask = torch.triu(torch.ones(T, T, device=q.device), diagonal=1).bool()
    scores = scores.masked_fill(mask, float('-inf'))

    attn = F.softmax(scores, dim=-1)
    return attn @ v

Why not just zero the entries?

Replacing future scores with 0 before softmax doesn't forbid them — exp(0) = 1, so they'd still receive mass. Using -inf sends exp(-inf) → 0, which is the only value softmax treats as "not an option."

What production code does differently

Build the mask once. Register it as a buffer (e.g. self.register_buffer("bias", tril)) and slice it to the current sequence length. Allocating a new mask per forward is O(T²) wasted memory.

Prefer F.scaled_dot_product_attention on any modern PyTorch — it fuses the mask + softmax + matmul and routes to Flash Attention on CUDA. Your implementation here is the educational version; the production call is one line with is_causal=True.

Watch the dtype. In mixed precision, softmax should run in float32 even if q/k/v are bfloat16. Otherwise the -inf masking can round to a finite negative value and break causality.
